Instructions

  • Start project Webapplication1 then start ConsoleApplication1, or in visual studio set the projects to multistart
  • The client will take screenshots and send it to the Wcf service. If the client running in debug mode events will not be fired, however all incomming events will display in the Consoleapplication.
  • The screenshot from the client will be placed in memory using MemoryMapped.
  • User events will be sent to WCF service using Ajax.
  • *If your running the client and the server on the same machine without two screens you will see a mirrory affect*

Only one client can use the service at a time due to the singelton instance pattern used in the WCF.

Im working on a WCF service that will use sessions based MemoryMappedfiles for all variables and resources this will make it possible to use one WCF with multiple clients.
